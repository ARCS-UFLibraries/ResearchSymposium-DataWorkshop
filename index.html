Workshop Goals

  1. Know the stages of a research data life‑cycle.
  2. Create a shareable, versioned data package using Google Drive.
  3. Leave a one‑page action plan for your own project.

Research Data Life‑Cycle

  • 1️⃣ Data Collection
  • 2️⃣ Data Processing (Cleaning / Transformation)
  • 3️⃣ Documentation and Metadata
  • 4️⃣ Data Preservation and Versioning
  • 5️⃣ Discovery (Indexing & DOI)
  • 6️⃣ Data Reuse or disposition (Archiving / Deletion)

Stage 1 – Generation (Data Collection)

Start with a raw data file (e.g., a CSV, an RDS, or an Excel spreadsheet). For the demo we’ll use a tiny sample CSV.

participant_id,value,date
001,24.3,2026‑06‑12
002,18.1,2026‑06‑12
003,22.0,2026‑06‑12

Stage 3 – Processing (Cleaning / Transformation)

Open the CSV in a tool of your choice (Excel, R, Python) and apply any cleaning steps. The output remains a new CSV.

  • Remove rows with missing values.
  • Convert units, if needed.
  • Rename columns for consistency.

Export the clean file as starter_data_clean.csv and keep both the raw and cleaned files.

Stage 2 – Documentation (Metadata)

Create a simple Dublin‑Core header for the file. We’ll build the XML by hand.

<?xml version="1.0" encoding="UTF-8"?>
<dc:RDF xmlns:dc="http://purl.org/dc/elements/1.1/">
  <dc:description>Laboratory experiment on plant photosynthesis.</dc:description>
  <dc:identifier>UF‑EX‑2026‑001</dc:identifier>
  <dc:creator>University of Florida, Dept. of Biology</dc:creator>
  <dc:format>text/csv</dc:format>
  <dc:date>2026-07-01</dc:date>
</dc:RDF>

Save this to experiment_01_dc.xml.

Stage 4 – Preservation (Versioning)

  1. Create a Google Drive folder called UF-Research-Data‑Demo.
  2. Upload:
    • experiment_01_raw.csv
    • experiment_01_clean.csv
    • experiment_01_dc.xml
  3. Create a Google Sheet named Version‑History with columns:
    FileVersionChangeTimestamp
    experiment_01_raw.csv0.1Initial upload2026‑07‑01
    experiment_01_clean.csv0.1Initial cleaning2026‑07‑01
    experiment_01_dc.xml0.1Metadata header2026‑07‑01
  4. Share the folder with read‑only link (no external sharing yet).

Stage 5 – Discovery (Indexing & DOI)

  1. Open the Google Drive folder in a browser.
  2. Click **+ New → File upload** and upload all three files again (or drag the folder into the Drive). Ensure the folder is still UF-Research-Data‑Demo.
  3. On the folder toolbar click Add accession (Zenodo button).
  4. Choose Open‑source, click **Generate DOI**.
  5. Copy the DOI URL (e.g., https://doi.org/10.5281/zenodo.1234567) and drop it into the next slide.

Stage 6 – Data Reuse (Archiving / Deletion)

When a dataset is no longer actively used:

  • Move the Google Drive folder to an archival location (e.g., UF Data Bank)
  • Keep the DOI and metadata for future reference.
  • Delete local copies to free disk space.

Policy: follow UF Acceptable data use’ guide.
